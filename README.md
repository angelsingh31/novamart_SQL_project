# NovaMart Retail Sales Analysis – SQL Skills Showcase
## 🚀 Overview
This project was created as a hands-on showcase of my SQL skills. Using a simulated retail dataset, I explored customer behavior, sales trends, and product performance while practicing advanced SQL concepts.
Think of it as a mini playground where I turned SQL practice into meaningful insights!

## 💡 Purpose
To demonstrate my ability to write complex queries
To extract actionable insights from data
To practice and showcase joins, aggregates, window functions, and subqueries

## 📊 Dataset & Tables
customers – Customer details
products – Product catalog with categories and prices
orders – Orders placed by customers
order_items – Products and quantities per order

## 🛠 Skills Practiced & Demonstrated
Joins & Subqueries – Combining multiple tables for insights
Aggregate Functions – SUM, COUNT, AVG
GROUP BY & HAVING – Summarizing data by categories
Window Functions – ROW_NUMBER(), RANK(), SUM() OVER()
Filtering & Sorting – Using WHERE, ORDER BY, LIMIT

## 🔍 Example Questions Explored
Who are the top customers by spending?
What are the best-selling products?
How do sales vary by month or city?
Which customers place the most orders?

## ⚡ How to Run
Clone this repository
Open PostgreSQL / pgAdmin
Run create_tables.sql → then insert_data.sql
Run queries.sql to explore the analysis
